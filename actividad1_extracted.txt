Detección de objetos con YOLOv5 
 
Contexto de la actividad 
En esta práctica iniciaremos el desarrollo de un sistema de visión artificial completo. 
El objetivo no es únicamente entrenar un modelo, sino construir la primera versión 
funcional que posteriormente será desplegada, evaluada y mejorada en las siguientes 
sesiones. 
En esta primera fase nos centraremos en el entrenamiento y validación del modelo 
base, que servirá como punto de partida para su integración en una aplicación real. 
 
Objetivos  
 
En esta actividad se desarrollará la primera versión de un detector de objetos 
personalizado mediante fine-tuning de YOLOv5. 
Al finalizar esta fase el alumno dispondrá de: 
• Un modelo entrenado para resolver un problema real definido por él mismo. 
• Un dataset estructurado y documentado. 
• Una evaluación objetiva del rendimiento del modelo. 
• Un modelo listo para ser utilizado en un sistema posterior de inferencia. 
Este modelo será utilizado en la siguiente fase de la asignatura para su integración en 
una aplicación. 
 
Pautas de elaboración

objetos. El profesor  te dará un dataset de ejemplo para detectar daños en paneles 
solares y, con base en eso, tienes que definir tu propio problema, obtener tu propio

1. Plantea un problema para resolver que se pueda hacer mediante la detección de 
objetos. 
2. Busca o crea tu propio dataset. Dicho dataset tiene que estar anotado en el 
formato que usa YOLO. 
• Si decides buscar un dataset, puedes utilizar repositorios como, por ejemplo, 
Kaggle (https://www.kaggle.com/). 
• Si decides crear tu dataset, puedes utilizar una herramienta como, por ejemplo, 
Make Sense (https://www.makesense.ai/), para hacer las anotaciones sobre las 
imágenes.  
3. Accede al repositorio https://github.com/ultralytics/yolov5 donde se encuentra el 
modelo.  
 
4. Recomiendo que uses Google Colab (te puedes conectar a una instancia local si 
necesitas usar tu estructura de archivos o más potencia) o un Jupyter Notebook 
local (pero tendrás que configurar muchas dependencias). Se asume a partir de 
ahora el uso de Google Colab. 
 
5. Abre en Colab el YOLOv5 Tutorial. 
 
6. Ejecuta el clone del git de YOLOv5 que está justo al principio, en la sección Setup. 
 
7. Sube tu dataset a Colab. Puedes subirlo como .zip y descomprimir dentro. Ten en 
cuenta la siguiente estructura de ficheros: 
• data/images/train: imágenes para entrenar el modelo. 
• data/images/val: imágenes para validar el modelo. 
• data/label/train: anotaciones de las imágenes train (.txt).

8. Usa de base el fichero yolov5/data/coco128.yaml. Lo tendrás disponible después 
de hacer el clone de YOLOv5. Sobre el contenido de coco128.yaml indica tu

• Haz un train con el archivo personalizado.yaml. Para ello , en el apartado 3 del 
YOLOv5 Tutorial modifica la instrucción: !python train.py --img 640 --batch 16 -
-epochs 3 --data personalizado.yaml --weights yolov5s.pt –cache 
 
9. Si tienes problemas de memoria ejecutando la instrucción del paso anterior , 
prueba a bajar el batch_size. Ten en cuenta que en la línea anterior ejecutas solo

subas el número de epochs al menos a cincuenta, para obtener mejores 
resultados.  
 
10. En yolov5/runs/exp encontrarás la salida de los resultados de validación. Además, 
en la ejecución verás métricas de evaluación. Crea un informe de validación 
usando la información que consideres relevante.  
 
11. Haz inferencia de los datos. Sube nuevas imágenes (o vídeos) y, apoyándote en las 
instrucciones de la sección «1. Inference», del tutorial de YOLOv5 haz tus propias 
detecciones de objetos.

Continuidad del proyecto 
El modelo desarrollado en esta actividad no constituye el resultado final del proyecto. 
En las siguientes sesiones se trabajará sobre este mismo modelo para: 
• Integrarlo en una aplicación capaz de procesar nuevas imágenes o vídeo. 
• Analizar su rendimiento en un entorno más realista. 
• Estudiar cómo mejorar el modelo mediante nuevos datos y ajustes. 
• Comprender el ciclo completo de desarrollo de sistemas basados en IA. 
Por tanto, es importante conservar tanto el dataset como los pesos generados 
durante el entrenamiento.
